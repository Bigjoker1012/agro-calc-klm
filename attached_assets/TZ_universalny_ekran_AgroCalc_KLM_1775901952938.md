# ТЗ для исполнителя: универсальный экран расчета AgroCalc КЛМ

## Задача
Разработать универсальный экран расчета для приложения AgroCalc КЛМ, который поддерживает несколько типов консервантов в одном интерфейсе без разделения на отдельные экраны под каждый продукт.[file:1][file:2]

## Принцип
Экран должен быть единым для всех препаратов, а различия в логике расчета реализуются через динамически отображаемые поля, правила дозирования и расчетные формулы.[file:1][file:2]

## Почему выбран такой подход
- Пользователь начинает не с продукта, а с сырья и условий заготовки.[file:1][file:2]
- SilKorm Pro и EGALIS Ferment используют разные расчетные модели, но опираются на общую входную структуру: культура, масса, влажность, способ внесения.[file:1][file:2]
- Такой подход позволит добавлять новые препараты без создания новых экранов.[file:1][file:2]

## Общая структура экрана
### 1. Блок выбора режима
В верхней части экрана разместить переключатель режима:
- Авто
- SilKorm Pro
- EGALIS Ferment

#### Логика режимов
- **Авто**: приложение само подбирает препарат по сырью и правилам дозирования.[file:1][file:2]
- **SilKorm Pro**: расчет выполняется только по правилам SilKorm Pro.[file:1]
- **EGALIS Ferment**: расчет выполняется только по правилам EGALIS Ferment.[file:2]

### 2. Общие поля ввода
Эти поля должны отображаться для всех режимов:
- Культура / сырье.
- Масса, т.
- Влажность, %.
- Способ внесения.
- Производительность, т/ч.

### 3. Условные поля
#### Для SilKorm Pro
Показывать дополнительные поля, если выбран SilKorm Pro или если Авто выбрал SilKorm Pro:
- Режим послойного внесения (если используется траншея):
  - выкл;
  - 75/100/125 %.[file:1]
- Тип внесения:
  - через комбайн;
  - опрыскиватель;
  - ручное внесение.[file:1]

#### Для EGALIS Ferment
Показывать дополнительные поля, если выбран EGALIS Ferment или если Авто выбрал EGALIS Ferment:
- Вариант пакета:
  - 50 г;
  - 200 г.[file:2][web:30]
- Схема рабочего раствора:
  - 50 л на пакет;
  - 200 л на пакет.[file:2]
- При необходимости: отображать поле “Количество пакетов” как вычисляемое, не редактируемое значение.[file:2][web:30]

## Порядок блоков на экране
1. Заголовок приложения.
2. Переключатель режима (Авто / SilKorm Pro / EGALIS Ferment).
3. Выбор сырья.
4. Параметры заготовки.
5. Способ внесения.
6. Условные поля по продукту.
7. Кнопка “Сформировать расчет”.
8. Карточка результата.
9. Кнопки действий: сохранить, PDF, отправить.

## Справочники и связь с базой
Экран должен работать не на жестко вшитых значениях, а на справочниках из базы.

### Необходимые таблицы
- `Продукты`
- `Культуры_и_сырье`
- `Правила_дозирования`
- `Пакеты_EGALIS`
- `Журнал_расчетов`

### Правило выбора записи из таблицы `Правила_дозирования`
Поиск нужной строки должен выполняться по набору условий:
- код продукта;
- код культуры;
- диапазон влажности;
- способ внесения;
- активность записи.

Если найдено несколько строк:
- выбирать наиболее точную по влажности;
- при отсутствии точного совпадения брать ближайшее допустимое правило;
- при невозможности расчета выводить сообщение “Нет подходящего правила дозирования”.

## Логика расчета
### 1. SilKorm Pro
Использовать нормы из базы, соответствующие инструкции:[file:1]
- злаковые травы — 5 л/т;
- бобовые и однолетние травы — 6 л/т;
- кукуруза — 3 л/т;
- плющеное зерно 35–45 % — 3–4 л/т;
- плющеное зерно 30–35 % — 4–5 л/т;
- плющеное зерно 25–30 % — 5–6 л/т.

#### Формулы SilKorm Pro
- Дозировка на тонну = значение из правила дозирования.[file:1]
- Общий объем препарата, л = масса, т × выбранная дозировка, л/т.[file:1]
- Настройка дозатора, л/ч = производительность, т/ч × дозировка, л/т.[file:1]
- Настройка дозатора, л/мин = (л/ч) / 60.

#### Режим послойного внесения
Если активирован режим 75/100/125 %:
- нижний слой = 75 % от нормы;
- средний слой = 100 % от нормы;
- верхний слой = 125 % от нормы.[file:1]

### 2. EGALIS Ferment
Использовать базовую норму 2 г/т и упаковки 50 г и 200 г, соответствующие 25 т и 100 т при базовой дозе.[file:2][web:30]

#### Формулы EGALIS Ferment
- Норма продукта = 2 г/т.[file:2][web:30]
- Масса препарата, г = масса, т × 2.
- Тонн на пакет:
  - 50 г = 25 т;[web:30]
  - 200 г = 100 т.[web:30]
- Количество пакетов = округление вверх (масса, т / тонн на пакет).
- Вода, л = количество пакетов × выбранный объем воды на пакет (50 или 200 л).[file:2]
- Рабочий раствор, л/т = общий объем воды / масса, т.[file:2]

## Логика режима Авто
### Цель
Режим Авто должен подбирать продукт по таблице правил и показывать наиболее подходящее решение на основе введенных параметров.[file:1][file:2]

### Минимальная логика для MVP
- Сначала ищется правило для SilKorm Pro по выбранной культуре и влажности.[file:1]
- Для культур и сценариев, покрытых EGALIS Ferment, допускается отдельная таблица приоритетов или ручная настройка приоритетов администратором.[file:2][web:26][web:29]
- Если авто-правило не найдено, вывести пользователю понятное сообщение и предложить ручной выбор продукта.

### Что показать пользователю в режиме Авто
В карточке результата обязательно вывести:
- выбранный продукт;
- причину выбора (например: “Кукуруза, влажность 32 %, базовое правило дозирования”);
- дозу;
- общий объем;
- способ внесения;
- дополнительные параметры по препарату.[file:1][file:2]

## Карточка результата
После нажатия “Сформировать расчет” в нижней части экрана должен появляться блок результата.

### Поля результата
Общие:
- Продукт.
- Культура / сырье.
- Масса.
- Влажность.
- Дозировка на тонну.
- Общий расход препарата.
- Комментарий / рекомендация.

Для SilKorm Pro:
- Объем препарата, л.[file:1]
- Настройка дозатора, л/ч и л/мин.[file:1]
- Если есть цена — итоговая стоимость.

Для EGALIS Ferment:
- Масса препарата, г.[file:2][web:30]
- Количество пакетов.[web:30]
- Вариант пакета.
- Объем воды.
- Рабочий раствор, л/т.[file:2]
- Если есть цена — итоговая стоимость.

## Кнопки действий после расчета
Под карточкой результата вывести:
- Сохранить в историю.
- Скачать PDF.
- Отправить клиенту.

## Экран истории
История должна хранить расчеты, выполненные пользователем.

### Для каждой записи хранить
- дата и время;
- клиент;
- продукт;
- культура;
- масса;
- влажность;
- дозировка;
- общий объем;
- статус синхронизации.

### На экране истории показать
- список карточек расчетов;
- поиск/фильтр;
- пустое состояние;
- открытие расчета повторно.

## Экран настроек
Настройки должны содержать:
- источник данных;
- кнопку синхронизации с Google Sheets;
- статус синхронизации;
- справочник препаратов;
- правила дозирования.

### Статус синхронизации должен показывать
- активно / нет связи;
- дата последней синхронизации;
- количество несинхронизированных записей;
- последняя ошибка, если была.

## Офлайн-режим
Приложение должно работать после первой загрузки без интернета.

### В офлайне должно быть доступно
- открытие приложения;
- работа главного экрана;
- расчет;
- сохранение в историю;
- просмотр локальной истории;
- просмотр локальных справочников.

### При появлении сети
- отправка новых расчетов на сервер;
- синхронизация в Google Sheets;
- обновление справочников.

## UI/UX требования
- Мобильный приоритет.
- Крупные элементы управления.
- Простые подписи на русском языке.
- Не более одного основного действия на экран — “Сформировать расчет”.
- Визуально выделенный результат.
- Понятные сообщения об ошибках и отсутствии правил.

## Что не делать
- Не делать отдельные самостоятельные экраны для каждого консерванта.
- Не дублировать общие поля в разных местах.
- Не вшивать жестко дозировки в интерфейсный код.
- Не завязывать расчет только на интернет-соединение.

## Критерии приемки
Результат считается принятым, если:
- один экран поддерживает минимум два продукта: SilKorm Pro и EGALIS Ferment;[file:1][file:2]
- поля экрана адаптируются под выбранный продукт;
- расчет выполняется корректно по таблице правил;
- результат выводится внизу экрана после нажатия кнопки;
- расчет можно сохранить в историю;
- логика работает без интернета после первой загрузки;
- значения берутся из базы, а не из тестовой обманки.
